# Phase 16 Plans 01/03/05: MOT Export + Tracking Plausibility + Tactical Validation Summary

**One-liner:** MOT Challenge CSV export, zero-ground-truth physical plausibility checks (5 tests), and Metrica Sports tactical cross-validation via convex-hull/stretch-index comparison.

## Tasks Completed

### Task 1 — Plan 16-01: export_mot_csv() pipeline function

Added `export_mot_csv()` to `backend/pipeline/src/analytics/__init__.py` (commit `ba04634`).

- MOT Challenge CSV format: `frame,id,bb_left,bb_top,bb_width,bb_height,conf,-1,-1,-1`
- Frame numbers are 1-indexed per MOT convention
- Categories default to `['players', 'referees', 'goalkeepers']` (ball excluded)
- Track ID global uniqueness via offsets: players=0, goalkeepers+1000, referees+2000
- Lines sorted by frame then track ID
- Added to `__all__` and imported + called in `all.py` after `export_tracks_json`
- Output: `{video_name}_mot.csv` alongside existing analytics exports

### Task 2 — Plan 16-03: tracking_plausibility.py

Created `backend/evaluation/tracking_plausibility.py` (commit `8e8e440`).

Five physical plausibility checks, all zero-ground-truth:

| Check | Threshold | Fallback |
|-------|-----------|----------|
| Speed outliers | >36 km/h sustained 2+ frames | pixel coords if no pitchX/Y |
| Position jumps | >5m in one frame | pixel coords |
| Player count | Outside [18, 26] per frame | N/A |
| Pitch bounds | Outside 105m x 68m rectangle | Skip gracefully if no pitch coords |
| Ball speed | >150 km/h | pixel coords |

`compute_plausibility_summary()` returns `plausibility_rate` = fraction of frames passing ALL applicable checks.

Outputs: `plausibility_report.json`, `plausibility_summary.tex`, `speed_distribution.pdf`, `player_count_timeseries.pdf`

### Task 3 — Plan 16-05: tactical_validation.py

Created `backend/evaluation/tactical_validation.py` (commit `338a746`).

- Downloads Metrica Sports Game 1 home+away CSVs from GitHub raw URLs via `urllib`
- `parse_metrica_csv()` converts normalized [0,1] Metrica coordinates to metres (× 105, × 68)
- `compute_tactical_from_metrica()` computes 5 metrics matching our pipeline: convex hull area, stretch index, inter-team distance, possession fraction (territory-based), defensive line height
- `compare_distributions()` maps our summary values against Metrica P25/median/P75 range
- Verdicts: `in range` / `above` / `below`
- Outputs: `tactical_comparison.json`, `tactical_comparison.tex` LaTeX table, `tactical_distributions.pdf`
