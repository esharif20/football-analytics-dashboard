### Visualization-Specific Color Contracts

**Heatmap (VIZ-01)**
- Cell color = team color from `useTeamColors()` hook (`TEAM1_HEX` / `TEAM2_HEX`)
- Cell opacity encodes intensity (existing approach, no change)
- Team 1 base: `oklch(0.6 0.2 25)` (red) тАФ CSS custom property `--color-team-1`
- Team 2 base: `oklch(0.55 0.18 200)` (blue) тАФ CSS custom property `--color-team-2`
- Fix: SVG z-index must be above `pitch-container::after` vignette (z-index: 3). SVG must render at z-index 4 or higher.

**Pass Network (VIZ-02)**
- Node color: uses `.player-node` CSS class with `--player-color` CSS custom property set to team color
- Node size: 28├Ч28px diameter, 2px white border тАФ matches existing `.player-node` spec exactly
- Edge color: team color at constant opacity (no arrowheads тАФ `<marker>` elements removed entirely)
- Edge stroke-width range: 0.15тАУ1.2 (existing scale, encodes pass frequency/weight)
- No `markerEnd` attribute on any `<path>` or `<line>` element

**Ball Trajectory (VIZ-03)**
- Trail gradient: white at ~15% opacity at tail тЖТ bright amber (`oklch(0.85 0.15 90)`) at head
- Tail segment: `rgba(255, 255, 255, 0.15)` or `color-mix(in oklch, white 15%, transparent)`
- Head segment: `oklch(0.85 0.15 90)` at full opacity тАФ matches `--color-ball` token
- Gradient is multi-segment: opacity + hue shift along the polyline from tail to head
- Ball head pulse animation: preserve existing `<animate>` elements unchanged (`ball-glow` keyframes)
- Trail rendering: use windowed trail segments only (not all points simultaneously)

## SVG Stacking Contract

All three visualizations use inline SVG inside `.pitch-container`. The stacking order must be:

| Layer | z-index | Element |
|-------|---------|---------|
| 1 | 1 | `pitch-container::before` (stripe texture) |
| 2 | 2 | SVG pitch lines (field markings) |
| 3 | 3 | `pitch-container::after` (vignette) тАФ current problem layer |
| 4 | 4+ | SVG data layer (heatmap cells, pass edges, trajectory) |
| 5 | 10 | `.player-node` elements |
| 6 | 15 | `.ball-marker` element |

Fix approach (Claude's discretion per CONTEXT.md D-01): either promote the data SVG to z-index 4, or reduce the `::after` vignette to z-index 2 and accept minor visual change. Evaluate both options in the codebase тАФ choose the one that avoids unintended side effects across all three viz tabs.
